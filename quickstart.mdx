---
title: "Quickstart: run Ladder and proxy your first URL"
description: "Get Ladder running locally with Docker in minutes. Proxy a URL through the browser UI, try the JSON API, and fetch raw HTML — all in one guide."
---

Ladder is a self-hosted HTTP proxy you run on your own machine or server. This guide gets you from zero to proxying your first URL in a few commands. The fastest path is Docker — no build step required.

<Warning>
  If you plan to expose your Ladder instance to the internet, set the `USERPASS` environment variable to enable Basic Auth. Without it, anyone who can reach your instance can proxy arbitrary content through your server.
</Warning>

<Steps>
  <Step title="Run Ladder with Docker">
    Pull and start the official container, pointing it at the community ruleset:

    ```bash
    docker run -p 8080:8080 -d \
      --env RULESET=https://raw.githubusercontent.com/everywall/ladder-rules/main/ruleset.yaml \
      --name ladder \
      ghcr.io/everywall/ladder:latest
    ```

    Ladder starts on port `8080`. The ruleset is loaded from the community repository on startup.
  </Step>

  <Step title="Open the proxy UI">
    Navigate to [http://localhost:8080](http://localhost:8080) in your browser. You'll see a URL input form. Enter any URL and press Enter — Ladder fetches the page and rewrites links so you can continue browsing through the proxy.
  </Step>

  <Step title="Proxy a URL directly">
    Append any full URL to the proxy address to fetch it immediately without the form:

    ```
    http://localhost:8080/https://www.example.com
    ```

    <Tip>
      Create a browser bookmark with this URL to send any page you're viewing through Ladder instantly:

      ```javascript
      javascript:window.location.href="http://localhost:8080/"+location.href
      ```
    </Tip>
  </Step>

  <Step title="Try the JSON API">
    Use the `/api` endpoint to fetch a page and inspect the request and response headers:

    ```bash
    curl -X GET "http://localhost:8080/api/https://www.example.com"
    ```

    The response is a JSON object containing the page body and header arrays. See the [API reference](/api/overview) for the full response schema.
  </Step>

  <Step title="Try RAW mode">
    Fetch the plain HTML source without any URL rewriting:

    ```bash
    curl "http://localhost:8080/raw/https://www.example.com"
    ```

    RAW mode returns the HTML as plain text — useful for parsing or processing programmatically.
  </Step>
</Steps>
