Ladder is a self-hosted HTTP proxy built for developers, researchers, and QA engineers who need to test and analyze how websites serve content under different conditions. It lets you simulate different client environments — such as Googlebot or custom browsers — and observe how paywalls, CORS policies, and content delivery behave.

## How Ladder works

When you send a request through Ladder, it applies your configured request modifications (headers, user agent, cookies), fetches the target site, then applies response modifications (HTML rewriting, regex rules, code injection) before returning the result to your browser or API client.

```
client → Ladder (apply RequestModifications) → website → Ladder (apply ResultModifications) → client
```

## Get started

<Steps>
  <Step title="Run with Docker">
    Pull and run the official container image with a public ruleset:

    ```bash
    docker run -p 8080:8080 -d \
      --env RULESET=https://raw.githubusercontent.com/everywall/ladder-rules/main/ruleset.yaml \
      --name ladder \
      ghcr.io/everywall/ladder:latest
    ```
  </Step>
  <Step title="Open the proxy UI">
    Navigate to [http://localhost:8080](http://localhost:8080) in your browser. You'll see a URL input form where you can enter any URL to proxy it through Ladder.
  </Step>
  <Step title="Proxy a URL directly">
    Append any URL to the proxy address to fetch it immediately:

    ```
    http://localhost:8080/https://www.example.com
    ```
  </Step>
  <Step title="Configure your ruleset">
    Create a `ruleset.yaml` file to define per-domain rules for header manipulation, HTML injection, and more. See [Ruleset configuration](/configuration/ruleset) for the full schema.
  </Step>
</Steps>

<Warning>
  If your Ladder instance will be publicly accessible, enable Basic Auth by setting the `USERPASS` environment variable. Without it, anyone with access to your instance can proxy arbitrary content through your server.
</Warning>
